The data structure is a way of organizing data in a computer so that it can be accessed effectively. Ther are a different way of organizing data in a computer. Like Array, Binary Tree, Graph, Linked List, Binary Search Tree, Matrix, Stack, Heap, Misc, Queue, Hashing, Advance Data Structure. We can use the best method that suits the program.




  • Arrays - An array is a collection of items stored at contiguous memory locations. The idea to store multiple items of the same type together. This makes it easier to calculate the position of each element by simply adding an offset to a base value.
  • Linked list - Linked List is a linear data structure, in which the element ae not stored at the continuous memory location. But the elements in a linked list are linked using pointers. A linked list consists of nodes where each node contains a data field and a reference link to the next node of the list.
  • Stack - A stack is a linear data structure that follows a particular order in which the operations are performed. The order may be LIFO(Last in First Out) or FILO(First In Last Out).
  • Queue - A Queue is a linear structure that follows a particular order in which the operations are performed. The order is First in First Out(FIFO). A good example of a queue is any queue consumers for a resource where the consumer that came first is served first. The difference between a stack and a queue is in removing. In a stack we remove the item the most recently added; in a queue, we remove the item that least recently added.
  • Binary Tree - Binary Tree Data Structure is like a tree whose element has at most 2 children is called a binary tree. Since each element in a binary tree can have only 2 children, we typically name them the left and right child. A binary tree node contains the following parts. 1. Data 2. Pointer to left child 3.Pointer to right child.
  • Binary Search Tree - The binary search tree is a node-based tree data structure which has the following properties: The left subtree of a node contains only nodes with keys lesser than the node’s key. The right subtree of a node contains only nodes with keys greater than the node’s key. The left and right subtree each must also be a binary search tree.
  • Heap Data Structure - A heap is a special tree-based data structure in which the tree is a complete binary tree. Generally, Heaps can be of two types:
    •  Max-Heap: In a max-heap, the key present at the root node must be greatest among the keys present at all of its children. The same property must be recursively true for all subtree in that binary tree.
    •  Min-Heap: In a min-heap, the key present at the root node must be minimum among the keys present at all of its children. The same property must be recursively true for all sub-tree in that Binary tree.
  • Hashing Data Structure - Hashing is an important data structure that is designed to use a special function called the hash function which is used to map a given value with the particular key for faster access of elements. The efficiency of mapping depends on the efficiency of the hash function used.
  • Graph Data Structure - A graph is a non-linear data structure consisting of nodes and edges. The nodes are sometimes also referred to as vertices and the edges are lines or arcs that connect any two nodes in the graph.
  • Matrix Archives - A matrix represents a collection of numbers arranged in an order of rows and columns. It is necessary to enclose the elements of a matrix in parentheses or brackets.

Programming paradigm is simply a pattern or style to write a program and it’s not about syntax it’s more than that. The programming paradigms can be broadly classified into two types imperative programming and declarative programming.

  1. Declarative Programming – It focuses on what to execute it define program logic but not detailed control flow. For example, if we need to repeat something for a number of times we just declare and repeat n number of times. We don’t need to write or bother about the detailed flow of the procedure.
  2. Imperative programming focuses on how to execute it define control flow as a statement. It changes the program state. Considering the same example mentioned above in imperical programming style, to repeat a task n times we write the detailed flow of the program with the help of a loop.
    Other paradigms which comes under either of these are:
  • Functional Programming - In this method we use only functions. We organize code in functions with clearly defined tasks.
  • Procedural Programming - They are set of subroutine they can modify variables and change the state fo the program. Here you don’t organize code in logical groups or logical pieces of data. You don’t use objects or entities to structure your code in the end you just execute step after step.
  • Object-Oriented Programming - It is all about manipulating and accessing objects, here you organize your data methods in classes and then in objects. Everything in your application gets represented as class and then in end as objects. Here you organize your code in logical entities.

    • But most of the time we use multi-paradigm programming. For example in a warehouse, we use object-oriented for keeping stocks and inventories and functional programming for mathematical anaylisis and calculations to see what product is doing well.
